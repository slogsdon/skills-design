You're still designing for yourself.

Software interfaces went UX → DX → AX. The primary consumer of your specs, configs, project notes, and documentation is now an AI agent operating autonomously between sessions. The drift you've been blaming on prompts is somewhere you haven't looked yet.

§ 02 / Three observations

The thing I kept adjusting wasn't the problem.

01 / The pattern

UX → DX → AX.

Each shift redesigned the same surfaces around a different reader. The latest reader is an agent operating autonomously between sessions, filling ambiguity with the most statistically probable answer.

02 / The symptom

When sessions drift, the prompt is rarely the problem.

I refined prompts for six months. The corrections list looked nearly identical to month two. The drift was coming from the specs and project notes — everything the agent reads to infer intent.

03 / The fix

Write for the agent reading it cold.

Stop assuming continuity that doesn't exist. Inference at the edges of the session is exactly what causes the drift.
